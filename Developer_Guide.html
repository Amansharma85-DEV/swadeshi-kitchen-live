Swadeshi Kitchen - Customization Guide

This guide explains how to copy, customize, and deploy this cloud kitchen web application template for other brands or projects.

1. Project Structure Map

File / Folder Path Description What you edit here
vite.config.ts Vite build configuration file. Change the base path to match your new GitHub repository name.
src/App.tsx Main application router entry point. Manage page routing (uses HashRouter for static page routing safety).
src/Home.tsx The main customer-facing landing page. Contact fallbacks, page sections, scroll animations, WhatsApp templates.
src/lib/store.ts Local storage state management file. Initial default menu items, default timings, prices, brand logo paths.
src/lib/firebase.ts Firebase connection logic. Connect your cloud database for syncing menu changes and order lists.
src/admin/Login.tsx Admin credentials check. Change login email/password validation check.
src/index.css Core styles and Tailwind styles. Configure button styles, fonts, core inputs, and custom scrollbars.

2. Step-by-Step Customization

Step A: Updating Base Path (For New GitHub Repo)

When deploying to a new GitHub repository (e.g. https://github.com/user/new-brand), update the base folder config:

vite.config.ts
export default defineConfig({
  base: '/new-brand/', // Replace 'new-brand' with your new repository name
  server: { ... }

Step B: Customizing default branding and logos

Change default menu prices, thali categories, and default branding configurations inside the storage script:

src/lib/store.ts
const defaultSettings: GlobalSettings = {
  hero: {
    title: 'Authentic Homestyle Food.',
    subtitle: 'Delicious dishes cooked fresh and delivered to your doorstep.',
    backgroundImage: 'https://images.unsplash.com/...'
  },
  contact: {
    email: 'hello@newbrand.com',
    phone: '+919999999999',
    address: 'Your Kitchen Address Here',
    openingHours: 'Mon-Sun: 8:00 AM - 11:00 PM'
  },
  brand: {
    name: 'New Brand Name',
    logo: '/new-brand/logo.png' // Matches the assets base path
  }
};

Step C: Editing WhatsApp order templates

Customize the invoice text layout that is sent to WhatsApp during customer checkout:

src/Home.tsx
const createWhatsAppMessage = (order: StoredOrder) => {
  const lines = order.items.map((item) => `- ${item.quantity} x ${item.name} (${formatCurrency(item.price * item.quantity)})`);
  return [
    '====================================',
    '       *NEW CUSTOMER ORDER*         ',
    '====================================',
    '',
    `*Customer:* ${order.customer.name}`,
    `*Phone:* ${order.customer.phone}`,
    `*Address:* ${order.customer.address}`,
    '',
    '*Items:*',
    ...lines,
    '',
    '------------------------------------',
    `*Total:* ${formatCurrency(order.totals.grandTotal)}`,
    `*Payment:* ${order.paymentMethod}`,
    '===================================='
  ].join('\n');
};

Step D: Changing Admin Login details

To update the admin login credentials, edit the validation checks inside the Login page controller:

src/admin/Login.tsx
// Look for this verification check:
if (email === 'admin@swadeshikitchen.com' && password === 'admin123') {
  localStorage.setItem('swadeshi-admin-auth', 'true');
  navigate('/admin');
}
💡 Fast Color Customization Tip

This template uses Tailwind CSS. To change the visual color scheme (e.g. from Orange to Blue), use a search-and-replace tool in your editor to swap all occurrences of orange classes (like bg-orange-600, hover:bg-orange-700, text-orange-700) to your new color (like blue classes: bg-blue-600, hover:bg-blue-700, text-blue-700).

3. Local Development & Deployment Commands

Run these commands inside your project terminal to test and publish changes:

# 1. Install dependencies
npm install

# 2. Run local development server
npm run dev

# 3. Compile and build the production bundle
npm run build

# 4. Push updates to main branch
git add .
git commit -m "update customizations"
git push origin main

# 5. Build and Deploy live to GitHub Pages
npm run deploy
